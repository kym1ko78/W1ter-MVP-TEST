# План доработки confirm-панели удаления

## Goal
Сделать появление confirm-панели удаления еще более плавным и убрать текст `Удаление...` с confirm-кнопки во время запроса.

## Assumptions
- Речь идет о текущем компоненте `apps/web/components/confirm-dialog.tsx`.
- Логика удаления уже работает корректно, меняется только UX.
- Под "более плавным" имеется в виду более мягкий и естественный enter/exit, без ощущения резкого всплытия.
- Вместо текста `Удаление...` во время pending state нужно сохранить более спокойный вид кнопки.

## Current State
Сейчас confirm-dialog:
- уже анимируется через fade + slide + scale
- выглядит лучше прежнего, но пользователь хочет еще более мягкое появление
- при pending state меняет текст кнопки на `Удаление...`
- это визуально дергает кнопку и делает окно менее спокойным

## Approach
Сделать две параллельные доработки:
1. смягчить motion за счет timing / easing / амплитуды движения
2. стабилизировать confirm-кнопку во время pending state без смены ее текста

## Plan

### Phase 1: Смягчить enter animation
- Пересмотреть текущие параметры анимации:
  - duration
  - easing
  - начальный `translate`
  - начальный `scale`
- Сделать движение менее резким:
  - уменьшить стартовый сдвиг
  - сделать scale-переход менее заметным
  - при необходимости чуть увеличить duration
- Основная цель:
  - панель должна не "впрыгивать", а мягко проявляться
- Output / exit criterion:
  - окно удаления визуально появляется спокойнее текущей версии

### Phase 2: Смягчить exit animation
- Проверить, не слишком ли быстро dialog исчезает по `Отмена` или после confirm.
- Если нужно:
  - сделать fade-out чуть длиннее
  - снизить резкость обратного scale/translate
- Важно сохранить быстрый UX, но убрать ощущение обрыва.
- Output / exit criterion:
  - закрытие confirm-panel выглядит естественно и не схлопывается резко

### Phase 3: Убрать текст `Удаление...` с кнопки
- В `apps/web/components/confirm-dialog.tsx` перестать подменять label confirm-кнопки на `Удаление...`.
- Оставить основной label, например `Удалить`, даже в pending state.
- При этом сохранить визуальную защиту от повторного нажатия:
  - `disabled`
  - уменьшенная opacity
  - при желании мягкий spinner или просто disabled state без смены текста
- Рекомендация:
  - сначала самый чистый вариант: тот же label + disabled state
- Output / exit criterion:
  - confirm button больше не дергает layout сменой текста

### Phase 4: Стабилизировать ширину и визуальный ритм кнопок
- Проверить, не меняется ли баланс кнопок в footer dialog из-за pending state.
- При необходимости:
  - задать более стабильный padding / min-width для confirm button
- Это особенно полезно, если раньше кнопка визуально расширялась от `Удаление...`.
- Output / exit criterion:
  - footer dialog остается визуально стабильным до, во время и после confirm

### Phase 5: Проверить влияние на UX и accessibility
- Убедиться, что во время pending state:
  - confirm button disabled
  - cancel button ведет себя предсказуемо
  - overlay click и Escape не ломают логику
- Проверить, не выглядит ли отсутствие `Удаление...` как "ничего не происходит".
- Если потребуется, использовать только очень мягкий визуальный индикатор:
  - opacity
  - subtle cursor state
  - без текстовой замены
- Output / exit criterion:
  - pending state понятен, но визуально спокойнее

### Phase 6: Проверить тестовую стабильность
- Убедиться, что изменения анимации не ломают e2e.
- Проверить, что confirm-кнопка остается доступной по тем же `data-testid`.
- Если понадобится, оставить текущие локаторы без изменения, чтобы не плодить лишние тестовые правки.
- Output / exit criterion:
  - motion и pending-state не делают UI e2e flaky

### Phase 7: Финальная визуальная оценка
- Проверить вручную:
  - удаление сообщения
  - удаление чата
- Оценить:
  - стало ли окно еще мягче
  - не стало ли оно слишком медленным
  - выглядит ли confirm-кнопка спокойнее без `Удаление...`
- Output / exit criterion:
  - диалог подтверждения стал визуально чище и деликатнее

## Testing and Validation

- Проверить confirm flow
- Проверить pending state без текста `Удаление...`
- Прогнать:
  - `pnpm --filter @repo/web typecheck`
  - `pnpm verify:web`
  - `pnpm test:ui:e2e:auto`

## Risks and Dependencies
- Риск: слишком медленная анимация сделает UX вязким.
- Mitigation:
  - увеличивать плавность умеренно, а не радикально.

- Риск: без текста `Удаление...` pending state станет неочевидным.
- Mitigation:
  - оставить disabled + opacity и при необходимости легкий визуальный индикатор без смены label.

- Риск: изменение timing сделает тесты менее стабильными.
- Mitigation:
  - сохранить короткую анимацию и не менять тестовые `data-testid`.
