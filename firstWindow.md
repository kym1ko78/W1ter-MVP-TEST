## Goal
Упростить первый auth-экран:
- заменить бейдж `live` на `Alfa`
- убрать подпись `Welcome back`
- полностью очистить левую белую колонку, оставив в ней только логотип `W1ter`

## Assumptions
- Работа касается только auth-экрана в [auth-screen.tsx](C:\Users\User\Desktop\Project\apps\web\components\auth-screen.tsx).
- Нужно менять только контент и layout первого экрана, без изменения логики входа/регистрации.
- Правая темная карточка с формой остается основной визуальной частью экрана.
- Левую колонку не нужно удалять целиком: нужно оставить ее как чистое пространство с одним логотипом.

## Constraints
- Не ломать responsive layout auth-экрана.
- Не ломать существующие формы и переходы между `/login` и `/register`.
- Не трогать логику submit, validation и auth flow.
- Сохранить общий визуальный стиль текущего auth-screen, но сделать его заметно минималистичнее.

## Architecture / Approach
Изменение локализуется в одном компоненте:
- [auth-screen.tsx](C:\Users\User\Desktop\Project\apps\web\components\auth-screen.tsx)

Работа делится на 3 независимых блока:
1. упростить header правой темной панели
2. очистить левую белую колонку
3. проверить, что после удаления контента layout не развалился и не остались пустые декоративные зазоры

### Phase 1: Найти и зафиксировать текущую структуру auth-screen
- Найти в [auth-screen.tsx](C:\Users\User\Desktop\Project\apps\web\components\auth-screen.tsx):
  - блок `Welcome back`
  - бейдж `live`
  - левую белую колонку с большим hero-контентом

  - вертикальные отступы
  - `justify-between`
  - вложенные секции внутри левой колонки
- Output / exit criterion:

### Phase 2: Заменить `live` на `Alfa`
- В правой темной панели найти текущий бейдж `live`.
- Заменить текст на `Alfa`.
- Проверить, нужно ли сохранить тот же uppercase-стиль:
  - если да, оставить визуальный стиль, но текст будет `Alfa`/`ALFA` в зависимости от существующего casing
- Output / exit criterion:
  - вместо `live` отображается `Alfa`

### Phase 3: Убрать подпись `Welcome back`
- В том же верхнем блоке правой панели удалить:
  - `<p class="...">Welcome back</p>`
- После удаления проверить, не осталось ли лишнего `mt-3`, `gap` или пустого отступа перед заголовком:
  - `Войдите в рабочее пространство.`
- При необходимости подтянуть заголовок выше.
- Output / exit criterion:
  - верхняя часть правой панели выглядит чисто, без “Welcome back” и без пустой дырки сверху

### Phase 4: Полностью очистить левую белую колонку, оставив только логотип
- В левой колонке удалить весь контент, кроме верхнего логотип-блока:
  - навигацию `Features / Realtime / Security`
  - `Web Messenger MVP`
  - большой hero-заголовок
  - описательный текст
  - карточки `Flow / Realtime`
  - карточки `Fast entry / Calm UI / Single flow`
- Оставить только:
  - значок `W`
  - текст `W1ter`
- Output / exit criterion:
  - в левой колонке остается только логотип, все остальные блоки удалены

### Phase 5: Пересобрать layout левой колонки после удаления контента
- После очистки проверить, нужен ли текущий `justify-between`.
- Скорее всего, после удаления почти всего контента колонку нужно упростить:
  - убрать лишнюю вертикальную растяжку
  - оставить более простой alignment для логотипа
- Возможно потребуется:
  - заменить `flex flex-col justify-between` на более простой контейнер
  - скорректировать `min-height`/padding, чтобы логотип не выглядел потерянным
- Output / exit criterion:
  - левая колонка выглядит как чистое, намеренно пустое пространство с логотипом, а не как “сломанный блок после удаления контента”

### Phase 6: Проверить баланс между пустой левой колонкой и правой формой
- После очистки левой части экран станет гораздо минималистичнее.
- Нужно проверить:
  - не стала ли левая колонка слишком пустой визуально
  - не перекосился ли общий баланс в сторону правой панели
  - не нужно ли слегка подправить ширины колонок или внутренние отступы
- Вероятный вывод:
  - базовую сетку можно оставить
  - но нужно убедиться, что пустая левая зона выглядит осознанной
- Output / exit criterion:
  - экран смотрится минималистично, а не “недорендеренным”

### Phase 7: Проверить `/login` и `/register`
- Убедиться, что упрощенный layout одинаково хорошо работает:
  - на экране входа
  - на экране регистрации
- Особенно проверить:
  - не разъехался ли верхний блок правой панели
  - не влияет ли удаление левого контента на общую высоту/scroll
- Output / exit criterion:
  - оба auth-сценария выглядят корректно

### Phase 8: Финальная полировка
- Проверить, не остались ли:
  - пустые wrapper-блоки
  - лишние `mt`, `mb`, `gap`
  - декоративные классы у уже удаленных секций
- Упростить JSX, чтобы auth-screen был чище не только визуально, но и структурно.
- Output / exit criterion:
  - auth-screen после правки минимален и логически чист
