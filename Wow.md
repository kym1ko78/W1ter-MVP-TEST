## Goal
Сместить правую темную auth-панель в центр доступной области и добавить на оставшийся белый фон декоративные полоски, чтобы пустое пространство выглядело оформленным, а не просто белым полотном.

## Assumptions
- Работа идет в [auth-screen.tsx](C:\Users\User\Desktop\Project\apps\web\components\auth-screen.tsx) и, при необходимости, в [globals.css](C:\Users\User\Desktop\Project\apps\web\app\globals.css).
- Под “перемести это в середину” имеется в виду визуально центрировать темную правую карточку внутри своей колонки, а не растягивать ее к краям.
- Полоски должны появиться именно на оставшейся белой области экрана, не перекрывая форму входа.
- Логика auth-form не меняется.

## Constraints
- Не ломать `/login` и `/register`.
- Не ломать responsive behavior.
- Не превращать полоски в навязчивый шум; они должны быть декоративным фоном.
- Не затрагивать form controls и их доступность.

## Architecture / Approach
Задача делится на две визуальные части:
1. перепозиционирование темной панели
2. оформление белого фона полосатым паттерном

Вероятнее всего:
- центрирование панели можно решить изменением layout внутри [auth-screen.tsx](C:\Users\User\Desktop\Project\apps\web\components\auth-screen.tsx)
- полоски лучше оформить через CSS-класс или псевдоэлемент в [globals.css](C:\Users\User\Desktop\Project\apps\web\app\globals.css), чтобы не раздувать JSX

## Plan
### Phase 1: Зафиксировать текущую структуру auth-layout
- Проверить текущую сетку:
  - белая левая колонка
  - правая секция `auth-dark-panel`
  - внутренняя темная карточка
- Определить, что сейчас удерживает темную панель ближе к верхнему краю или не по центру:
  - `order`
  - `px/py`
  - `flex`
  - `justify-between`
  - `mt-8`, `mt-6` внутри темной панели
- Output / exit criterion:
  - понятно, на каком уровне нужно центрировать: секция, внутренний контейнер или form-card

### Phase 2: Центрировать темную панель в своей области
- Изменить layout правой секции так, чтобы темная панель:
  - визуально сидела по центру колонки
  - не прилипала к верхней части
  - сохраняла аккуратные outer paddings
- Проверить:
  - достаточно ли `items-center` / `justify-center`
  - не нужно ли уменьшить верхние служебные отступы внутри самой панели
- Output / exit criterion:
  - темная auth-card сидит по центру композиции, а не воспринимается смещенной

### Phase 3: Подтянуть внутренний контент панели под новое центрирование
- После смещения панели проверить внутреннюю композицию:
  - заголовок
  - бейдж `Alfa`
  - form-card
- Возможно, придется уменьшить:
  - `mt-8`
  - `mt-6`
  - `justify-between`
  чтобы панель выглядела собранно и не тянулась пустотами
- Output / exit criterion:
  - внутренняя композиция темной панели уравновешена после центрирования

### Phase 4: Спроектировать полосатый фон для белой части
- Решить, какой именно паттерн нужен:
  - вертикальные тонкие полосы
  - диагональные мягкие полосы
  - очень светлые фоновые линии
- Для текущего минималистичного экрана лучше всего:
  - очень светлые, редкие, полупрозрачные полосы
  - без сильного контраста
- Output / exit criterion:
  - выбран конкретный визуальный тип полосок, который не спорит с интерфейсом

### Phase 5: Реализовать полоски через CSS, а не через лишний JSX
- Добавить отдельный фон-класс для белой области:
  - например через `linear-gradient`
  - или через `::before/::after`
- Привязать этот класс к нужной обертке auth-screen.
- Убедиться, что полоски:
  - остаются под контентом
  - не перекрывают форму
  - не мешают читаемости логотипа
- Output / exit criterion:
  - белый фон получил аккуратный полосатый паттерн

### Phase 6: Проверить баланс между темной панелью и новым белым фоном
- После добавления полосок оценить:
  - не стали ли они слишком заметными
  - не перетягивают ли внимание от формы
  - не выглядят ли случайным декоративным шумом
- При необходимости уменьшить:
  - opacity
  - размер шага
  - контраст полос
- Output / exit criterion:
  - фон выглядит намеренно и стильно, но вторично по отношению к форме

### Phase 7: Проверить `/login` и `/register`
- Убедиться, что:
  - темная панель центрирована одинаково для обоих экранов
  - полоски одинаково уместны на login и register
  - layout не ломается на меньших ширинах
- Output / exit criterion:
  - auth-screen стабилен в обоих режимах

### Phase 8: Финальная полировка
- Проверить:
  - не осталось ли старых отступов, из-за которых панель все еще кажется сдвинутой
  - не выглядят ли полоски обрезанными или слишком резкими
- При необходимости сделать последнюю микроитерацию по:
  - padding
  - alignment
  - opacity/size полос
- Output / exit criterion:
  - экран выглядит визуально завершенным и чище, чем до правки

## Risks and Dependencies
- Риск: полоски будут слишком заметными и начнут спорить с формой.
- Mitigation:
  - делать их очень светлыми и декоративными, а не акцентными.

- Риск: центрирование темной панели сломает вертикальные отступы внутри нее.
- Mitigation:
  - после изменения внешнего positioning отдельно проверить внутренние `mt` и `justify-*`.

- Риск: на mobile полоски или центрирование дадут неожиданный layout.
- Mitigation:
  - проверить responsive behavior и при необходимости ограничить эффект для desktop breakpoint.
