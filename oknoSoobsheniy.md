## Objective
Сделать так, чтобы страница чата помещалась в высоту окна браузера без прокрутки всей страницы, а скроллилась только переписка внутри окна сообщений.

## Assumptions
- Цель: убрать вертикальный scroll у всего сайта в режиме чата.
- Скролл должен остаться только у:
- списка чатов слева
- истории сообщений справа
- Header чата и composer внизу должны быть всегда видимы в пределах экрана.
- Поведение нужно сохранить и на desktop, и на mobile.
- Меняется только layout/scroll behavior, без изменения backend.

## Current Cause
Сейчас страница растет по высоте окна из-за сочетания:
- в `apps/web/components/chat-shell.tsx` используется `min-h-screen`
- внутренний grid тоже использует `min-h-[calc(100vh-24px)]`
- в `apps/web/components/conversation-view.tsx` секция чата имеет `min-h-[70vh]`
- message list уже имеет `overflow-y-auto`, но внешний контейнер все равно может стать выше viewport

Из-за этого браузер скроллит всю страницу, а не только историю сообщений.

## Implementation Plan

### Phase 1: Зафиксировать высоту shell по viewport
- В `apps/web/components/chat-shell.tsx` заменить модель `min-height` на жесткую высоту viewport.
- Вместо `min-h-screen` перейти на `h-screen` или лучше `h-[100dvh]` для более корректного поведения на мобильных браузерах.
- Для внешнего контейнера добавить `overflow-hidden`, чтобы страница не разрасталась вниз.
- Внутренний grid тоже перевести с `min-h-*` на `h-full`.

Output / exit criterion:
- общий shell чата занимает ровно высоту окна, без роста страницы вниз

### Phase 2: Разрешить внутренним колонкам нормально ужиматься
- Для sidebar и правой колонки добавить `min-h-0`.
- Для правой секции, в которой рендерится `children`, тоже явно задать `min-h-0`.
- Это важно, потому что во flex/grid layout без `min-h-0` дочерний scroll-контейнер часто не начинает скроллиться и вместо этого растит родителя.

Output / exit criterion:
- левая и правая колонка перестают толкать страницу вниз

### Phase 3: Перестроить окно переписки как “header + scroll area + composer”
- В `apps/web/components/conversation-view.tsx`:
- заменить `min-h-[70vh]` на `h-full min-h-0`
- оставить `section` как `flex flex-col`
- Зафиксировать:
- header как `flex-none`
- composer form как `flex-none`
- message list как `flex-1 min-h-0 overflow-y-auto`
- Проверить, чтобы только message list получал скролл, а не весь `section`.

Output / exit criterion:
- header и composer всегда видны
- прокрутка идет только внутри message list

### Phase 4: Привести sidebar к той же модели
- В `apps/web/components/chat-shell.tsx` сделать sidebar тоже `h-full min-h-0`.
- Верхний профиль и поиск оставить фиксированными.
- Область списка чатов сделать отдельным scroll-контейнером:
- `flex-1 min-h-0 overflow-y-auto`
- Убрать зависимость от `lg:max-h-[calc(100vh-270px)]`, если она больше не нужна после перевода всего layout на `h-full`.

Output / exit criterion:
- слева скроллится только список чатов, а не вся колонка

### Phase 5: Проверить placeholder и пустые состояния
- В `apps/web/components/chat-placeholder.tsx` убедиться, что пустой экран чата растягивается на доступную высоту колонки, а не требует внешнего page scroll.
- При необходимости дать placeholder `h-full min-h-0`.

Output / exit criterion:
- даже без выбранного чата правая колонка не ломает высоту страницы

### Phase 6: Мобильное поведение
- На mobile проверить, не ломается ли высота из-за адресной строки браузера.
- Если `100vh` ведет себя нестабильно, использовать `100dvh`.
- Проверить:
- нет лишнего page scroll
- composer доступен
- история сообщений скроллится нормально
- Если надо, отдельно скорректировать padding верхнего/нижнего контейнера.

Output / exit criterion:
- на мобильном браузере скролл страницы тоже не доминирует над внутренним скроллом переписки

## Testing and Validation
- Открыть чат с большим количеством сообщений:
- сайт не должен прокручиваться целиком вниз
- прокручиваться должна только история сообщений
- Проверить, что header чата остается на месте
- Проверить, что composer остается видимым внизу
- Проверить длинный список чатов:
- прокручивается только список слева
- Проверить пустой чат / placeholder
- Проверить desktop
- Проверить mobile width
- Проверить после отправки сообщения:
- layout не прыгает
- scroll area остается внутри окна переписки
- При необходимости прогнать `Playwright`, если текущий сценарий зависит от стабильной видимости composer

## Risks and Dependencies
- Риск: после перевода на `overflow-hidden` может случайно перестать скроллиться и message list, и sidebar.
- Митигация: обязательно добавить `min-h-0` на промежуточные контейнеры.
- Риск: `100vh` на мобильных браузерах даст неправильную высоту.
- Митигация: использовать `100dvh`.
- Риск: существующие `min-h-[70vh]` и `max-h-[calc(...)]` классы будут конфликтовать с новой схемой.
- Митигация: убрать старые высотные ограничения, а не наслаивать новые поверх них.
- Риск: при авто-растущем composer окно переписки может уменьшаться слишком агрессивно.
- Митигация: оставить message list как `flex-1 min-h-0`, чтобы он корректно отдавал место composer.
