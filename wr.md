# План сохранения placeholder-экрана до явного выбора чата

## Goal
Сделать так, чтобы placeholder-экран `Choose a thread ...` не исчезал автоматически, когда у пользователя появляются чаты, и оставался видимым до тех пор, пока пользователь сам не нажмет на конкретный чат в списке слева.

## Assumptions
- Речь идет о placeholder-экране в chat shell, который сейчас показывается в пустом состоянии.
- Сейчас исчезновение placeholder связано не с явным выбором пользователя, а с авто-переходом на первый чат.
- Пользователь хочет более предсказуемое поведение:
  - чаты могут уже существовать
  - но пока пользователь не кликнул на конкретный чат, placeholder должен оставаться
- После явного клика по чату поведение должно стать обычным: открывается диалог.

## Current State
Сейчас в логике chat shell есть автонавигация:
- если маршрут `/chat`
- и в списке уже есть чаты
- и `currentChatId` отсутствует
- приложение автоматически делает redirect на первый чат

Из-за этого:
- placeholder-экран исчезает сам
- пользователь не успевает остаться на обзорном состоянии
- появление первого чата автоматически меняет правую панель

## Approach
Убрать auto-redirect на первый чат и оставить placeholder как дефолтное состояние маршрута `/chat`.
Переход к конкретному диалогу должен происходить только если:
- пользователь кликнул по чату в sidebar
- либо явно открыт маршрут `/chat/[chatId]`

## Plan

### Phase 1: Найти авто-редирект на первый чат
- Проверить `apps/web/components/chat-shell.tsx`.
- Найти `useEffect`, который при наличии `chatsQuery.data` и отсутствии `currentChatId` делает `router.replace(/chat/{firstId})`.
- Подтвердить, что именно этот эффект и заставляет placeholder исчезать автоматически.
- Output / exit criterion:
  - найден точный участок логики, который нужно убрать или изменить

### Phase 2: Убрать авто-открытие первого чата
- Изменить поведение так, чтобы маршрут `/chat` оставался валидным самостоятельным состоянием даже при наличии чатов.
- Placeholder должен продолжать рендериться, если:
  - пользователь находится на `/chat`
  - конкретный `chatId` не выбран
- Output / exit criterion:
  - наличие чатов больше не приводит к автоматическому переходу в первый диалог

### Phase 3: Сохранить ручной переход по клику на чат
- Проверить, что карточки чатов слева по-прежнему ведут на `/chat/[chatId]`.
- Убедиться, что после клика:
  - placeholder исчезает
  - открывается выбранный диалог
- Output / exit criterion:
  - правый экран меняется только после явного выбора пользователя

### Phase 4: Проверить создание нового direct chat
- Отдельно проверить сценарий создания нового чата через поиск пользователя.
- Решить, какое поведение нужно здесь:
  - либо после создания direct chat сразу открывать его
  - либо тоже оставаться на placeholder до явного выбора
- Так как пользователь формулирует требование как "пока я не нажму на конкретный чат", правильнее выбрать:
  - после создания чат появляется в списке
  - placeholder остается
  - пользователь сам открывает его кликом
- Если это слишком сильно ломает текущий UX, можно отдельно зафиксировать компромисс.
- Output / exit criterion:
  - логика создания чата согласована с новым правилом выбора

### Phase 5: Проверить deep-link сценарии
- Убедиться, что если пользователь сразу открывает URL вида `/chat/[chatId]`, то:
  - placeholder не показывается
  - открывается нужный чат
- Это важно, чтобы не сломать прямые ссылки и reload внутри открытого диалога.
- Output / exit criterion:
  - placeholder сохраняется только для `/chat`, а не для конкретного `chatId`

### Phase 6: Проверить пустое и непустое состояние sidebar
- Пустое состояние:
  - чатов нет
  - placeholder виден
- Непустое состояние:
  - чаты есть
  - placeholder тоже виден, пока нет выбора
- После клика по чату:
  - placeholder исчезает
  - справа открывается conversation view
- Output / exit criterion:
  - оба сценария ведут себя предсказуемо и согласованно

### Phase 7: Обновить тесты и статус
- Если текущие e2e предполагают авто-открытие первого чата, обновить их под новое поведение.
- Проверить как минимум:
  - после логина на `/chat` placeholder может оставаться
  - после клика по chat-list-item открывается диалог
- При необходимости обновить `implementation-status.md`.
- Output / exit criterion:
  - тесты и документация соответствуют новому UX

## Risks and Dependencies
- Риск: удаление авто-редиректа изменит текущий сценарий после логина сильнее, чем ожидается.
- Mitigation:
  - проверить отдельно login flow и открытие существующих чатов.

- Риск: после создания direct chat пользователь может ожидать мгновенного перехода внутрь.
- Mitigation:
  - явно выбрать и зафиксировать желаемое поведение до финальной реализации.

- Риск: e2e тесты завязаны на старый auto-open flow.
- Mitigation:
  - обновить тесты под новый ручной выбор чата.
