## Goal
Удалить декоративные элементы из пустого состояния чата: `chat-scribble-mark` и оба `chat-orbit-ring`, чтобы placeholder стал полностью чистым и минималистичным.

## Assumptions
- Удаляются только декоративные `div`, не влияющие на логику, данные или навигацию.
- Основной контейнер placeholder, заголовок и остальная структура остаются без изменений.
- После удаления нужно убедиться, что на экране не осталось лишнего пустого слоя `pointer-events-none absolute inset-0`, если он больше не нужен.

## Plan
### Phase 1: Подтвердить источник декоративных элементов
- Открыть `chat-placeholder.tsx`.
- Найти блок, где рендерятся:
- `chat-scribble-mark right-0 top-0`
- `chat-orbit-ring -bottom-10 -left-10 h-32 w-32 opacity-90`
- `chat-orbit-ring bottom-10 right-12 h-20 w-20 border-[14px] opacity-70`
- Убедиться, что это именно текущий placeholder, который видит пользователь.
- Выходной результат фазы: точное место правки подтверждено.

### Phase 2: Удалить декоративные `div`
- Удалить все перечисленные декоративные элементы из JSX.
- Проверить, нужен ли после этого внешний wrapper:
- если `pointer-events-none absolute inset-0` больше пустой, удалить и его;
- если внутри него еще что-то остается, оставить только нужное.
- Выходной результат фазы: placeholder не содержит этих графических украшений.

### Phase 3: Проверить композицию экрана
- Убедиться, что после удаления декоративных фигур:
- заголовок не “заваливается” визуально;
- placeholder не выглядит сломанным по отступам;
- не осталось странных пустых абсолютных слоев.
- Если нужно, слегка подправить spacing контейнера, но не вводить новый дизайн.
- Выходной результат фазы: пустое состояние выглядит чисто и аккуратно.

### Phase 4: Validation
- Прогнать `pnpm --filter @repo/web typecheck`.
- Прогнать `pnpm verify:web`.
- Визуально проверить placeholder в браузере:
- без выбранного чата;
- на обычной ширине;
- на узком экране.
- Выходной результат фазы: декоративные элементы удалены без регрессий.

## Risks and Dependencies
- Риск: можно удалить только внутренние `div`, но оставить пустой absolute-wrapper без смысла.
- Смягчение: после удаления проверить, остался ли wrapper нужен.

- Риск: placeholder может стать слишком пустым.
- Смягчение: сохранить основной заголовок и текущую базовую структуру без лишних изменений.
