## Goal
Убрать встроенную SVG-лупу из поля поиска, сохранив рабочее поведение поиска и аккуратную верстку input без лишнего пустого места слева.

## Assumptions
- Речь идет о декоративной иконке-лупе внутри search input, а не о функциональной кнопке поиска в верхней панели.
- Логика поиска, `data-testid`, placeholder и обработчики не должны измениться.
- После удаления иконки нужно проверить, не остались ли у input лишние левые отступы, рассчитанные под этот абсолютный SVG.

## Plan
### Phase 1: Найти источник SVG-лупы
- Найти JSX-узел с указанным SVG или соответствующий компонент-обертку, где эта иконка рендерится.
- Определить, к какому именно поисковому полю относится иконка:
- global search;
- message search;
- user search;
- другой input.
- Проверить, используется ли для этого input специальный `pl-*` отступ под иконку.
- Выходной результат фазы: точное место правки и список зависимых классов layout.

### Phase 2: Удалить иконку без поломки layout
- Удалить JSX/SVG-элемент лупы из соответствующего поля.
- Если input имел увеличенный левый padding под абсолютную иконку, уменьшить его до обычного проектного значения.
- Проверить, что placeholder и введенный текст теперь визуально начинаются с правильного места.
- Выходной результат фазы: иконка исчезла, а input выглядит естественно и не “провален” вправо.

### Phase 3: Проверить связанные состояния
- Убедиться, что удаление затронуло только декор и не сломало:
- фокус input;
- hover/focus-ring;
- очистку поля;
- поиск по Enter или по onChange.
- Если тот же styled input используется повторно в нескольких местах, проверить, не убрали ли иконку там, где она еще нужна.
- Выходной результат фазы: изменение локальное и предсказуемое.

### Phase 4: Validation
- Прогнать `pnpm --filter @repo/web typecheck`.
- Прогнать `pnpm verify:web`.
- Вручную проверить нужный input в браузере:
- пустое состояние;
- длинный ввод;
- focus;
- mobile/узкая ширина, если поле адаптивное.
- Выходной результат фазы: точечная UI-правка полностью безопасна.

## Risks and Dependencies
- Риск: можно удалить SVG, но забыть убрать `padding-left`, из-за чего поле будет выглядеть криво.
- Смягчение: после удаления иконки сразу проверить соседние классы контейнера и самого input.

- Риск: иконка может использоваться в переиспользуемом компоненте, и изменение заденет несколько экранов.
- Смягчение: сначала точно определить, где рендерится этот SVG, и править только нужное место.

## Next Actions

- Удалить иконку и выровнять отступы input.
- Прогнать `typecheck` и `verify:web`.
