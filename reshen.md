# План доработки confirm-окна удаления

## Goal
Сделать окно подтверждения удаления заметно плавнее при появлении и закрытии, а саму карточку диалога визуально мягче: не строго квадратной, а с аккуратно скругленными углами.

## Assumptions
- Речь идет о кастомном confirm-диалоге в `apps/web/components/confirm-dialog.tsx`.
- Базовая анимация уже есть, но пользователь хочет:
  - более мягкое, более заметное появление
  - чуть скругленные края у самой панели
- Логика удаления сообщения и чата не меняется.
- UX должен оставаться быстрым и не становиться "тормозным".

## Current State
Сейчас confirm-dialog:
- уже не появляется резко
- использует короткий fade + небольшой slide/scale
- визуально остается довольно строгим и почти прямоугольным
- ощущается недостаточно мягким по motion и по форме

## Approach
Доработать confirm-dialog в двух направлениях:
1. сделать motion чуть более выразительным, но коротким
2. смягчить форму панели через умеренное скругление углов и при необходимости чуть более мягкую тень

## Plan

### Phase 1: Уточнить целевое ощущение анимации
- Оставить анимацию быстрой, но сделать ее более естественной.
- Сместить текущий баланс:
  - меньше "мгновенного щелчка"
  - чуть больше ощущения всплытия
- Подобрать новую схему:
  - overlay: более плавный fade
  - panel: чуть более заметный fade + scale/translate
- Output / exit criterion:
  - есть обновленный motion target, который воспринимается мягче текущего

### Phase 2: Ослабить резкость enter/exit transition
- В `apps/web/components/confirm-dialog.tsx` пересмотреть:
  - duration
  - easing
  - величину начального сдвига/масштаба
- Скорее всего подойдет:
  - чуть длиннее duration, чем сейчас
  - более мягкая easing curve
  - меньший резкий jump между hidden и visible состояниями
- Проверить enter и exit отдельно:
  - open должен ощущаться мягко
  - close не должен резко схлопываться
- Output / exit criterion:
  - окно удаления появляется и исчезает заметно плавнее

### Phase 3: Сделать панель визуально менее квадратной
- Увеличить `border-radius` у dialog panel.
- Не делать слишком круглую карточку, а выбрать умеренный вариант.
- Сохранить стиль интерфейса:
  - темная панель
  - чистые линии
  - мягкие, но не "мультяшные" углы
- Output / exit criterion:
  - панель выглядит более аккуратно и мягко, без ощущения жесткого прямоугольника

### Phase 4: При необходимости смягчить тень и фон
- Проверить, не стоит ли немного доработать:
  - shadow
  - backdrop opacity
  - blur
- Цель:
  - диалог должен выглядеть всплывающим слоем
  - но не тяжелой черной плитой
- Если потребуется, слегка изменить:
  - shadow spread
  - shadow blur
  - overlay darkness
- Output / exit criterion:
  - движение и форма воспринимаются как единый мягкий слой

### Phase 5: Сохранить UX и доступность
- Проверить, что после изменения анимации и формы все еще работают:
  - `Escape`
  - click on overlay
  - loading state
  - кнопки `Отмена` / `Удалить`
- Проверить reduced motion fallback:
  - сильное движение не должно ломать UX у пользователей с ограничением анимаций
- Output / exit criterion:
  - dialog остается удобным и предсказуемым

### Phase 6: Проверить влияние на тесты и кликабельность
- Убедиться, что confirm-кнопка остается стабильно кликабельной после анимации.
- Если нужно, сохранить текущую тестовую устойчивость без изменения test flow.
- Проверить, что dialog считается visible в правильный момент.
- Output / exit criterion:
  - новые transition не делают e2e flaky

### Phase 7: Финальная визуальная проверка
- Проверить вручную оба сценария:
  - удаление сообщения
  - удаление чата
- Оценить:
  - стало ли окно появляться мягче
  - достаточно ли скруглены углы
  - не стало ли окно слишком медленным
- Output / exit criterion:
  - confirm-dialog визуально ближе к желаемому референсу

## Testing and Validation
- Проверить открытие confirm-dialog
- Проверить закрытие по `Отмена`
- Проверить закрытие после `Удалить`
- Проверить `Escape`
- Проверить desktop layout
- Прогнать:
  - `pnpm --filter @repo/web typecheck`
  - `pnpm verify:web`
  - `pnpm test:ui:e2e:auto`

## Risks and Dependencies
- Риск: если сильно увеличить duration, окно станет ощущаться медленным.
- Mitigation:
  - увеличивать анимацию умеренно, без ухода в тяжелый motion.

- Риск: слишком сильное скругление выбьется из стиля UI.
- Mitigation:
  - использовать умеренный радиус, а не максимально круглый modal.

- Риск: изменение opacity/scale сделает тесты flaky.
- Mitigation:
  - сохранить простую структуру dialog и проверять видимость после enter.

## Next Actions

- Увеличить `border-radius` панели
- При необходимости слегка смягчить тень/overlay
- Прогнать web checks и UI e2e
