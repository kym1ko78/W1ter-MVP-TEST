## Goal
Привести верхнюю панель чата к текущему черно-белому стилю проекта, не меняя уже добавленную функциональность кнопок поиска, звонка, меню и правого профиля.

## Assumptions
- Меняем только визуальный слой верхней панели и связанных с ней строк поиска/статуса, без пересборки логики поиска, меню и drawer.
- Основная палитра должна совпадать с существующим UI чатов: белый фон, черный/темно-серый текст, светлые бордеры, без синего Telegram-оттенка.
- Правая профильная панель в рамках этой задачи может остаться темнее или быть доведена позже отдельным шагом, если верхняя панель уже визуально согласована.

## Plan
### Phase 1: Audit текущей верхней панели
- Найти все участки в `conversation-view.tsx`, где новый header использует темно-синий фон, синие акценты и Telegram-подобные hover-состояния.
- Отдельно зафиксировать связанные элементы, которые визуально привязаны к header:
- строка статуса под заголовком;
- раскрывающаяся строка поиска сообщений;
- выпадающее меню по трем точкам;
- визуальные active/hover-состояния action-кнопок.
- Определить, какие цвета уже используются в проекте рядом с chat shell, composer, group panel и sidebar, чтобы не вводить новую палитру.
- Выходной результат фазы: список точек, которые нужно привести к общей теме без изменения поведения.

### Phase 2: Перенос панели в черно-белую систему
- Заменить фон header на светлый или нейтральный проектный фон, который уже используется в чате.
- Вернуть заголовку и подзаголовку проектную типографику:
- темный заголовок;
- muted-серый статус;
- без синих акцентных сообщений по умолчанию.
- Перестроить action-кнопки справа так, чтобы они выглядели как нативные для текущего интерфейса:
- светлый фон или прозрачный фон;
- тонкий черный/серый border;
- hover через затемнение/бордер, а не через синий fill;
- активное состояние через черный/белый контраст или мягкий нейтральный fill.
- Проверить, что панель не выбивается из composer и message area по радиусам, отступам и плотности.
- Выходной результат фазы: верхний bar визуально выглядит как часть текущего продукта, а не как заимствованный Telegram-блок.

### Phase 3: Стилизация связанных интерактивных элементов
- Привести раскрывающийся поиск сообщений к той же черно-белой теме:
- фон и border как у остальных input-блоков проекта;
- счетчик и кнопки навигации в нейтральной палитре;
- кнопка закрытия без синего акцента.
- Привести popover-меню по трем точкам к стилю существующих floating-панелей:
- светлый фон;
- черный/серый текст;
- мягкая тень;
- hover в духе текущих меню/кнопок.
- Проверить статусное сообщение после клика по звонку:
- если оно остается, перевести его из голубого в нейтральный проектный акцент;
- если выглядит чужеродно, заменить на более спокойный оттенок из существующей системы.
- Выходной результат фазы: все элементы вокруг header читаются как единый UI-блок.

### Phase 4: Согласование с правым профилем
- Оценить, конфликтует ли новая светлая верхняя панель с уже существующим правым drawer профиля.
- Если контраст получается слишком резким, смягчить переход:
- оставить drawer как есть, но привести кнопку открытия профиля к общей теме;
- либо слегка подправить только верхнюю область drawer, не уходя в большой редизайн.
- Убедиться, что открытие profile panel из обновленного header не выглядит как переход между двумя разными приложениями.
- Выходной результат фазы: header и profile trigger выглядят согласованно даже при разной глубине стилизации.

### Phase 5: Проверка и доводка
- Проверить desktop-сценарий:
- обычный direct chat;
- group chat;
- поиск сообщений открыт;
- меню открыто;
- профиль открыт.
- Проверить mobile/узкий layout, чтобы action-кнопки не ломали заголовок и не съедали статус.
- Прогнать `pnpm --filter @repo/web typecheck`.
- Прогнать `pnpm verify:web`.
- Выходной результат фазы: визуальный рефактор не ломает сборку, адаптив и базовые сценарии.

## Risks and Dependencies
- Риск: можно случайно перекрасить только header, но оставить поиск и menu в старой синей теме, из-за чего интерфейс будет выглядеть нецельным.
- Смягчение: править header, search row и menu как один визуальный комплект.

- Риск: слишком светлая панель может потерять визуальное отделение от списка сообщений.
- Смягчение: сохранить четкую нижнюю границу, тень не добавлять без необходимости, опираться на текущие border и spacing-паттерны проекта.

- Риск: активные состояния кнопок станут недостаточно заметными после удаления синего акцента.
- Смягчение: использовать контраст через черный fill, темный text и subtle neutral background.

- Зависимость: финальная стилизация должна опираться на уже существующие классы и цветовые решения внутри chat shell, а не на новую палитру.
