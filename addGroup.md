## Goal
Переделать текущий блок создания группы в drawer так, чтобы вход в сценарий выглядел как отдельный пункт действия `Создать группу` с иконкой, как на референсе, а не как сразу открытая форма с кнопкой внизу.

## Assumptions
- Меняем только UX-композицию блока создания группы, а не backend-логику создания чата.
- Цвета, бордеры, отступы и общий светлый стиль текущего drawer сохраняются.
- Сценарий должен остаться совместимым с уже существующими state и mutations в [chat-shell.tsx](C:\Users\User\Desktop\Project\apps\web\components\chat-shell.tsx).

## Plan
### Phase 1: Target interaction design
- Зафиксировать новый целевой паттерн:
  - в списке действий drawer появляется строка `Создать группу` с иконкой;
  - по клику открывается отдельное вложенное состояние внутри drawer;
  - форма создания группы показывается только после выбора этого пункта.
- Решить формат раскрытия:
  - inline-раскрытие внутри drawer;
  - или замена секции действий на форму с кнопкой “назад”.
- Для текущего проекта выбрать более простой и устойчивый вариант:
  - inline-переход внутри drawer с локальным state `groupComposerOpen`.
- Output / exit criterion:
  - есть ясное UX-решение, как пользователь попадает в форму создания группы.

### Phase 2: Drawer action row redesign
- Убрать из постоянного содержимого drawer сразу раскрытый блок `Новая группа`.
- В секции `Действия` добавить новый action-row:
  - иконка групп;
  - текст `Создать группу`;
  - hover/focus-состояния в том же стиле, что у `Мой профиль` и `Выйти из аккаунта`.
- Сделать эту строку визуально похожей на второй референс:
  - одна строка;
  - иконка слева;
  - текст справа;
  - без тяжелой плашки-кнопки по всей карточке.
- Output / exit criterion:
  - в drawer виден компактный пункт `Создать группу`, похожий на элемент меню.

### Phase 3: Group creation reveal state
- Добавить локальный state для режима создания группы:
  - `isGroupComposerOpen` или аналогичный.
- Настроить поведение:
  - клик по `Создать группу` открывает форму;
  - закрытие drawer сбрасывает это состояние;
  - успешное создание группы тоже сбрасывает состояние.
- Добавить понятный способ вернуться назад:
  - кнопка `Назад`;
  - или компактный заголовок формы с крестиком/chevron.
- При открытии формы можно:
  - либо прокрутить drawer к форме;
  - либо отрисовать ее прямо вместо списка действий.
- Output / exit criterion:
  - форма группы не загромождает drawer по умолчанию и открывается только по действию пользователя.

### Phase 4: Form layout restyling
- Пересобрать форму создания группы так, чтобы она выглядела ближе к текущему drawer, но по логике ближе к референсу:
  - отдельный заголовок секции;
  - поле названия;
  - поле участников;
  - выбранные участники;
  - результаты поиска участников;
  - вспомогательный текст;
  - финальная кнопка создания.
- Стилистически не менять палитру, но сделать композицию визуально легче:
  - меньше ощущения “карточка внутри карточки”;
  - более четкая иерархия;
  - кнопка создания как финальный шаг формы, а не главный элемент всего drawer.
- Если нужно, усилить визуальное отличие активного режима создания группы от обычного списка действий через:
  - подзаголовок;
  - разделитель;
  - плавное раскрытие.
- Output / exit criterion:
  - форма выглядит как вторичный workflow внутри drawer, а не как всегда открытый большой блок.

### Phase 5: Logic preservation
- Сохранить существующую логику:
  - `groupTitleDraft`;
  - `groupSearch`;
  - `selectedGroupMembers`;
  - `groupUserSearchQuery`;
  - `groupCreateError`;
  - `createGroupChatMutation`.
- Убедиться, что перенос UI не ломает:
  - создание пустой группы;
  - создание группы с участниками;
  - удаление выбранных участников;
  - показ ошибок;
  - сброс формы после успешного создания.
- Добавить сброс состояния `isGroupComposerOpen`:
  - при успешном создании группы;
  - при закрытии drawer;
  - при переходе по маршруту.
- Output / exit criterion:
  - поведение осталось прежним, изменился только способ входа в сценарий.

### Phase 6: Validation and polish
- Проверить визуально:
  - drawer в обычном состоянии;
  - drawer с открытым режимом создания группы;
  - hover/focus на строке `Создать группу`.
- Проверить интерактивно:
  - открыть drawer;
  - открыть создание группы;
  - закрыть и заново открыть drawer;
  - создать группу;
  - вернуться в список действий.
- Убедиться, что на мобильной ширине новая структура не разваливается.
- Прогнать:
  - `pnpm --filter @repo/web typecheck`
  - `pnpm verify:web`
- Output / exit criterion:
  - новый сценарий `Создать группу` выглядит как action item из меню и проходит проверки.

## Risks and Dependencies
- Риск: inline-раскрытие может визуально перегрузить drawer.
  - Mitigation: при открытии формы скрывать или сворачивать список действий выше.
- Риск: пользователь потеряет контекст, где он находится.
  - Mitigation: добавить компактный заголовок формы и явную кнопку возврата.
- Риск: сброс состояния после создания группы будет неполным.
  - Mitigation: централизованно сбрасывать режим при `onSuccess`, закрытии drawer и смене маршрута.
- Dependency: вся реализация сосредоточена в [chat-shell.tsx](C:\Users\User\Desktop\Project\apps\web\components\chat-shell.tsx), где уже находятся состояние drawer и логика group creation.
