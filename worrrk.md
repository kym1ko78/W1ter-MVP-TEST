# План возврата правой auth-колонки и удаления float-card блоков

## Goal
Вернуть правую темную колонку auth-экрана к прежнему виду, где она не занимает всю доступную высоту/ширину как полноразмерная панель, и удалить два плавающих блока `auth-float-card-top` и `auth-float-card-bottom`.

## Assumptions
- Меняем только auth-экран.
- Левая колонка с основным контентом может остаться в текущем edge-to-edge режиме, если это не конфликтует с результатом.
- Нужно вернуть именно прежнюю композицию правой колонки:
  - с внешними отступами внутри общего auth-layout
  - с внутренней темной карточкой, а не полноразмерной сплошной секцией
- Плавающие блоки `Message flow` и `Auth status` должны быть полностью убраны из JSX.

## Current State
Сейчас после последней правки:
- правая колонка стала полноразмерной и прилегает к границам своей области слишком плотно
- внутренняя темная секция стала `rounded-none` и `lg:min-h-screen`
- исчезло ощущение отдельной темной карточки
- два плавающих блока `auth-float-card-top` и `auth-float-card-bottom` все еще присутствуют и добавляют лишний визуальный шум

## Approach
Разделить задачу на две части:
1. вернуть правой колонке прежнюю карточную композицию
2. удалить оба декоративных float-card блока из JSX

При этом важно сохранить:
- рабочую форму входа/регистрации
- текущие пути `/login` и `/register`
- адаптивность экрана

## Plan

### Phase 1: Зафиксировать, что именно нужно вернуть в правой колонке
- Сравнить текущее состояние правой секции с предыдущей композицией.
- Определить, какие именно классы нужно вернуть:
  - внешние `px/py` у правой секции
  - внутренний `rounded-[30px]`
  - прежнюю минимальную высоту
  - отказ от `lg:min-h-screen` и полноразмерного растяжения
- Output / exit criterion:
  - есть конкретный набор классов, который возвращает правую колонку к прежнему виду

### Phase 2: Вернуть правой колонке карточную оболочку
- В `apps/web/components/auth-screen.tsx` пересобрать правую колонку так, чтобы:
  - секция снова имела внешний внутренний отступ
  - темная область снова выглядела как отдельная внутренняя карточка
  - вернулись скругления у самой темной панели
- При этом не откатывать лишние изменения слева, если они не мешают задаче.
- Output / exit criterion:
  - правая колонка снова выглядит как отдельная темная карточка, а не полноразмерная заливка

### Phase 3: Полностью удалить `auth-float-card-top`
- Удалить из JSX блок `Message flow`.
- Убедиться, что после этого не остается лишних отступов или пустых зон.
- Output / exit criterion:
  - верхний плавающий блок исчезает без следов в layout

### Phase 4: Полностью удалить `auth-float-card-bottom`
- Удалить из JSX блок `Auth status`.
- Проверить, не остается ли визуальная дыра в нижней части правой колонки.
- Output / exit criterion:
  - нижний плавающий блок исчезает без поломанной композиции

### Phase 5: Подправить композицию после удаления float-card
- После удаления двух плавающих блоков проверить:
  - как сидит основной form card
  - не слишком ли пусто в темной панели
  - не нужно ли чуть сместить форму или заголовок
- При необходимости локально подправить:
  - `mt-6` / `mt-8`
  - `justify-between`
  - внутренние padding
- Output / exit criterion:
  - правая колонка выглядит собранной и цельной без float-card блоков

### Phase 6: Проверить оба auth-сценария
- Проверить `login`
- Проверить `register`
- Убедиться, что:
  - правая колонка снова не полноразмерная
  - плавающих карточек больше нет
  - форма и текст остаются на своих местах
- Output / exit criterion:
  - задача решена одинаково для обоих auth-маршрутов

## Risks and Dependencies
- Риск: возврат правой колонки случайно откатит и полезные изменения по outer spacing.
- Mitigation:
  - вернуть только правую секцию, не трогая левую без необходимости.

- Риск: после удаления float-card темная колонка станет слишком пустой.
- Mitigation:
  - локально скорректировать spacing основного form card и заголовка.

- Риск: на mobile композиция изменится не так, как на desktop.
- Mitigation:
  - отдельно проверить адаптивное поведение после правки.
