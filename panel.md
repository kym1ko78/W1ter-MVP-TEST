# План переработки composer-панели под референс

## Goal
Переделать нижнюю панель ввода сообщения так, чтобы она была ближе к референсу:
- убрать кнопку `Очистить`
- заменить текстовую кнопку `Прикрепить файл` на иконку-скрепку
- сделать composer визуально ближе к компактной chat-input bar
- показывать кнопку отправки только после ввода сообщения или выбора файла

## Assumptions
- Изменения касаются frontend composer в окне чата.
- Базовая логика отправки сообщения и прикрепления файла уже работает.
- `Enter` и `Ctrl+Enter` должны сохраниться.
- Нужно приблизить интерфейс к референсу, но не обязательно пиксель-в-пиксель копировать Telegram-like bar.

## Current State
Сейчас composer в `apps/web/components/conversation-view.tsx`:
- большой и карточный
- имеет текстовую кнопку `Прикрепить файл`
- имеет отдельную кнопку `Очистить`
- кнопка `Отправить` всегда видна справа
- визуально больше похожа на desktop form block, чем на chat message bar

Пользователь хочет:
- убрать `Очистить` совсем
- заменить прикрепление файла на иконку-скрепку
- приблизить панель к более компактному, горизонтальному формату
- показывать отправку только когда реально есть что отправлять

## Approach
Разделить задачу на 3 части:
1. упростить структуру composer
2. заменить attach-button на icon-button
3. изменить поведение send-button на conditional visibility

## Plan

### Phase 1: Зафиксировать новую структуру composer
- Определить целевую компоновку:
  - слева кнопка-скрепка
  - по центру поле ввода
  - справа кнопка отправки, которая появляется только при наличии текста/файла
- Решить, какие элементы из текущей панели оставить:
  - textarea
  - character counter
  - helper text про `Enter / Ctrl+Enter` — возможно сохранить, но упростить или перенести
- Output / exit criterion:
  - есть четкая новая layout-схема панели ввода

### Phase 2: Убрать кнопку `Очистить`
- В `apps/web/components/conversation-view.tsx` полностью удалить кнопку `Очистить` и связанный с ней визуальный блок внизу composer.
- При этом сохранить внутреннюю возможность очистки через другие пользовательские действия:
  - выделение и удаление текста
  - удаление выбранного файла через preview control
- Output / exit criterion:
  - в интерфейсе больше нет отдельной кнопки `Очистить`

### Phase 3: Заменить текстовую кнопку файла на иконку-скрепку
- Убрать текстовую кнопку `Прикрепить файл`.
- Вместо нее добавить icon-button со скрепкой.
- Кнопка должна:
  - открывать тот же hidden file input
  - быть компактной
  - визуально стоять слева в composer, как в референсе
- Решить источник иконки:
  - inline SVG
  - либо существующая иконка в проекте, если уже есть
- Output / exit criterion:
  - attach action доступен через компактную кнопку-скрепку

### Phase 4: Сделать send button условно видимой
- Изменить текущую логику так, чтобы кнопка отправки:
  - скрывалась, когда нет текста и нет файла
  - появлялась, когда есть текст или выбран файл
- Проверить, как это сочетается с disabled-state:
  - если кнопка скрыта, disabled по сути уже не нужен в idle state
  - если запрос отправляется, кнопка остается видимой как loading/pending state
- Output / exit criterion:
  - send button не занимает место в пустом composer и появляется только по делу

### Phase 5: Упростить геометрию панели под референс
- Пересобрать внешний вид composer, чтобы он был ближе к chat bar:
  - меньше визуальной высоты
  - более горизонтальный характер
  - меньше ощущения "огромной карточки"
- Проверить:
  - внутренние padding
  - высоту textarea
  - расположение counter/helper text
- При необходимости:
  - счетчик сделать менее заметным
  - helper text про `Enter / Ctrl+Enter` упростить или встроить аккуратнее
- Output / exit criterion:
  - composer стал визуально ближе к референсу и меньше похож на form panel

### Phase 6: Сохранить attachment preview UX
- Убедиться, что после замены кнопки на скрепку не ломается preview выбранного файла.
- Проверить сценарии:
  - выбран файл без текста
  - выбран файл с текстом
  - файл удаляется до отправки
- При необходимости визуально подстроить preview под новый более компактный composer.
- Output / exit criterion:
  - attach flow по-прежнему работает и выглядит согласованно с новой панелью

### Phase 7: Сохранить keyboard behavior
- Проверить, что изменения панели не ломают:
  - `Enter` для отправки
  - `Ctrl+Enter` для новой строки
  - авто-рост textarea
- Особенно важно убедиться, что условная send-button не мешает submit flow.
- Output / exit criterion:
  - вся текущая keyboard-логика сохраняется

### Phase 8: Проверить визуальную логику пустого и непустого состояния
- Пустое состояние:
  - нет send button
  - видна скрепка
  - placeholder читаемый
- Непустое состояние:
  - появляется send button
  - composer остается собранным и не скачет хаотично
- Нужно добиться плавного и понятного переключения между этими состояниями.
- Output / exit criterion:
  - empty/non-empty composer выглядит аккуратно и предсказуемо

### Phase 9: Обновить тесты при необходимости
- Если e2e или test selectors завязаны на текущую кнопку `Прикрепить файл` или постоянную кнопку `Отправить`, обновить их.
- Сохранить или переименовать `data-testid` осознанно:
  - для attach button
  - для send button
- Output / exit criterion:
  - тесты соответствуют новой панели и остаются стабильными

## Testing and Validation
- Проверить пустой composer
- Проверить ввод текста
- Проверить появление send button после ввода
- Проверить отправку текста
- Проверить `Ctrl+Enter`
- Проверить attach через скрепку
- Проверить attach-only message
- Проверить mobile/desktop layout
- Прогнать:
  - `pnpm --filter @repo/web typecheck`
  - `pnpm verify:web`
  - при необходимости `pnpm test:ui:e2e:auto`

## Risks and Dependencies
- Риск: скрываемая кнопка отправки будет вызывать layout jump.
- Mitigation:
  - сделать появление аккуратным и зарезервировать понятную структуру панели.

- Риск: иконка-скрепка окажется слишком маленькой или неочевидной.
- Mitigation:
  - выбрать достаточно контрастный размер и hit area.

- Риск: текущий preview файла визуально не впишется в новую компактную панель.
- Mitigation:
  - при необходимости слегка переработать preview после основной смены layout.

- Риск: тесты завязаны на старую кнопку attach/send.
- Mitigation:
  - заранее проверить `data-testid` и обновить e2e только там, где реально нужно.
