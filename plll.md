# План удаления серого контурного рисунка с auth-экрана

## Goal
Убрать светлый серый контурный рисунок с первого экрана входа/регистрации, где вводится почта и пароль, чтобы фон auth-страницы стал чище.

## Assumptions
- Речь идет о декоративном фоне auth-сцены, а не о формах, полях ввода или контентных карточках.
- Нужно убрать только контурный рисунок, сохранив остальной layout auth-экрана.
- Остальные декоративные элементы auth-экрана можно оставить, если они не относятся к этому конкретному рисунку.

## Current State
Сейчас на auth-экране присутствует светлый серый контурный рисунок:
- он расположен на фоне за формой
- выглядит как несколько крупных светлых волнистых контуров
- визуально напоминает тот же декоративный паттерн, который уже убирался из chat area
- вероятнее всего задается через `auth-scene::after` в глобальных стилях

## Approach
Найти источник рисунка в auth-стилях и убрать только этот слой:
- либо удалить `::after` background-image
- либо убрать конкретный `background-image`/SVG
- либо обнулить opacity этого декоративного слоя только для auth-сцены

## Plan

### Phase 1: Локализовать источник рисунка на auth-экране
- Проверить связанные файлы:
  - `apps/web/app/globals.css`
  - auth layout / auth screen компоненты, если нужно
- Найти, какой именно CSS-слой рисует контуры:
  - `auth-scene::after`
  - `background-image`
  - SVG/data-uri
- Output / exit criterion:
  - точно найден слой, отвечающий за серый рисунок на auth-экране

### Phase 2: Выбрать минимальную правку
- Решить, что лучше:
  - полностью убрать псевдоэлемент
  - убрать только `background-image`
  - или оставить сам слой, но без рисунка
- Предпочесть самое локальное изменение, которое не затронет остальную стилистику страницы.
- Output / exit criterion:
  - выбран безопасный способ удаления рисунка без побочных эффектов

### Phase 3: Убрать контурный рисунок из auth-scene
- Удалить или отключить слой, который отвечает за эти линии.
- Сохранить:
  - фон страницы
  - форму входа/регистрации
  - остальные рабочие декоративные элементы, если они не мешают
- Output / exit criterion:
  - на экране логина/регистрации больше нет серого контурного рисунка

### Phase 4: Проверить, не пострадали ли соседние auth-элементы
- Убедиться, что после удаления рисунка не сломались:
  - auth layout
  - form card
  - dark panel / float cards / scribble-элементы, если они нужны
- Если рисунок был частью общего слоя, оставить только те части декора, которые не создают шум.
- Output / exit criterion:
  - auth-экран остается цельным, но чище визуально

### Phase 5: При необходимости слегка выровнять фон auth-экрана
- Проверить, не стал ли фон слишком пустым или визуально незавершенным.
- Если нужно, оставить мягкий gradient/light background без контуров.
- Output / exit criterion:
  - auth-screen после удаления рисунка выглядит чисто и аккуратно

### Phase 6: Проверить оба auth-сценария
- Проверить экран `login`
- Проверить экран `register`
- Убедиться, что на обоих больше нет контурного рисунка
- Output / exit criterion:
  - задача решена для обоих auth-экранов

## Testing and Validation
- Проверить страницу входа
- Проверить страницу регистрации
- Проверить, что форма и layout не сместились
- Прогнать:
  - `pnpm --filter @repo/web typecheck`
  - `pnpm verify:web`

## Risks and Dependencies
- Риск: контурный рисунок находится в общем auth-слое и его удаление заденет другие визуальные элементы.
- Mitigation:
  - убирать только конкретный background/псевдоэлемент, а не весь auth-layer.

- Риск: фон станет слишком пустым.
- Mitigation:
  - оставить базовый светлый gradient без контуров.

- Риск: правка заденет login и register по-разному.
- Mitigation:
  - отдельно проверить оба маршрута после изменения.

- Убрать его локально
- Проверить login/register
- Прогнать web checks
