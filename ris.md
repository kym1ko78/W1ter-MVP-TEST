# План удаления серого контурного рисунка с фона чата

## Goal
Убрать светлый серый абстрактный рисунок с белого полотна области переписки, чтобы фон чата стал чище и визуально спокойнее.

## Assumptions
- Речь идет о декоративном фоне внутри основной chat-area, а не о сообщениях, карточках или sidebar.
- Нужно убрать только этот контурный рисунок, не ломая остальной layout и не затрагивая функциональность чата.
- Остальная стилизация интерфейса может остаться как есть, если после удаления фон выглядит хорошо.

## Current State
Сейчас в основной области чата на светлом фоне есть декоративный контурный рисунок:
- он выглядит как светлые волнистые линии
- расположен позади сообщений
- визуально создает лишний шум на белом полотне
- вероятнее всего задается либо через отдельный декоративный элемент, либо через CSS background/overlay

## Approach
Сначала точно определить, откуда рисуется этот фон, затем убрать только этот слой:
- либо удалить соответствующий декоративный DOM-элемент
- либо убрать background-image / pseudo-element
- либо обнулить opacity/фон только для chat-surface, если рисунок приходит из CSS

## Plan

### Phase 1: Найти источник рисунка
- Проверить связанные файлы chat UI:
  - `apps/web/components/conversation-view.tsx`
  - `apps/web/components/chat-placeholder.tsx`
  - `apps/web/components/chat-shell.tsx`
  - `apps/web/app/globals.css`
- Найти, чем именно задается рисунок:
  - отдельным `div`
  - CSS-классами
  - `::before` / `::after`
  - `background-image`
  - SVG/data-uri
- Output / exit criterion:
  - точно понятно, какой слой отвечает за серый рисунок

### Phase 2: Оценить минимальную точку правки
- Решить, что лучше:
  - убрать конкретный декоративный элемент из JSX
  - или убрать только фоновый стиль из CSS
- Предпочтение отдать самому локальному и безопасному изменению.
- Output / exit criterion:
  - выбран минимальный и чистый способ удаления рисунка

### Phase 3: Убрать декоративный рисунок из chat-area
- Удалить или отключить слой, который рисует контурный фон.
- При этом сохранить:
  - белое полотно переписки
  - структуру header / message list / composer
  - текущий scroll и layout
- Output / exit criterion:
  - фон чата становится чистым, без серых линий

### Phase 4: Проверить, не пострадали ли другие состояния
- Убедиться, что удаление фонового рисунка не затронуло:
  - empty state
  - placeholder state
  - loading skeleton
  - sidebar
- Если декоративный слой использовался в нескольких местах, оставить его только там, где он нужен, и убрать только из conversation canvas.
- Output / exit criterion:
  - рисунок исчезает именно там, где мешал, без побочного урона другим экранам

### Phase 5: При необходимости слегка подправить фон после удаления
- После удаления проверить, не стал ли фон слишком пустым или визуально "плоским".
- Если нужно, оставить очень мягкий градиент/тон, но без контурного рисунка.
- Цель:
  - чистый фон
  - без декоративного шума
  - без ощущения незавершенности
- Output / exit criterion:
  - chat background выглядит чисто и аккуратно после удаления рисунка

### Phase 6: Проверить визуально оба основных сценария
- Проверить чат с сообщениями
- Проверить пустой чат / placeholder
- Убедиться, что сообщения теперь читаются на более чистом фоне
- Output / exit criterion:
  - результат соответствует задаче и визуально лучше исходного варианта

## Testing and Validation
- Проверить открытый чат с несколькими сообщениями
- Проверить область переписки на desktop
- Проверить, что layout не сдвинулся
- Прогнать:
  - `pnpm --filter @repo/web typecheck`
  - `pnpm verify:web`
  - при необходимости `pnpm test:ui:e2e:auto`

## Risks and Dependencies
- Риск: рисунок используется общим классом для нескольких экранов.
- Mitigation:
  - сначала точно локализовать источник и убирать только нужный слой.

- Риск: после удаления фон станет слишком пустым.
- Mitigation:
  - при необходимости оставить очень мягкий однотонный/градиентный фон без контуров.

- Риск: правка CSS случайно затронет placeholder или sidebar.
- Mitigation:
  - проверять изменения именно в conversation area и в соседних состояниях.
