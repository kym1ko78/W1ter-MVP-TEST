## Goal
Убрать полосатый/полосоподобный рисунок с белого фона auth-экрана и заменить его на более чистое, красивое и спокойное оформление, где темные акценты остаются, но больше не читаются как полоски.

## Assumptions
- Меняем только белый фон auth-сцены на `/login` и `/register`.
- Центральная темная auth-панель, логотип `W1ter` и внутренняя белая форма уже стоят в нужной композиции.
- Пользователь хочет более чистый фон без полос, но с темным визуальным характером.
- Основная реализация должна остаться в [globals.css](C:\Users\User\Desktop\Project\apps\web\app\globals.css), чтобы не усложнять [auth-screen.tsx](C:\Users\User\Desktop\Project\apps\web\components\auth-screen.tsx).

## Plan
### Phase 1: Убрать зависимость от полосатого паттерна
- Полностью отказаться от текущего решения, где белый фон строится через вертикальный repeating/background pattern.
- Проверить, что после отключения полос композиция не станет слишком пустой.
- Сохранить фон как отдельный декоративный слой у `.auth-stage-surface`, чтобы дальше можно было спокойно подобрать более удачный стиль.
- Output / exit criterion:
  - полосатый мотив больше не участвует в фоне auth-сцены

### Phase 2: Определить новый тип темного декора
- Выбрать более спокойную альтернативу полоскам:
  - мягкие темные пятна
  - крупные размытые органичные формы
  - редкие темные волнообразные акценты по краям
- Сразу исключить решения, которые снова будут восприниматься как полосы или сетка.
- Оставить декоративность второго плана:
  - фон должен быть красивым
  - но не “рисовать на себе интерфейс”
- Output / exit criterion:
  - выбран новый фон без полосатого характера

### Phase 3: Собрать новый фон в одном контролируемом CSS-слое
- В [globals.css](C:\Users\User\Desktop\Project\apps\web\app\globals.css) пересобрать `.auth-stage-surface::before`.
- Использовать простой и управляемый фон:
  - `radial-gradient`
  - `linear-gradient`
  - при необходимости один SVG-слой, но без повторяющегося полосатого ритма
- Избегать избыточной сложности:
  - чем проще слой, тем легче его потом донастроить
- Output / exit criterion:
  - новый фон живет в одном CSS-источнике и легко регулируется

### Phase 4: Сохранить темный характер без визуальной тяжести
- Подобрать темные оттенки:
  - графит
  - угольно-серый
  - полупрозрачный почти черный
- Следить, чтобы темные формы:
  - были заметны
  - не делали белый фон грязным
  - не спорили с темной центральной панелью
- При необходимости ослабить фон через opacity и мягкие края.
- Output / exit criterion:
  - фон выглядит темным по характеру, но чистым по восприятию

### Phase 5: Сбалансировать фон вокруг центральной панели
- Проверить, как новый фон работает:
  - слева от панели
  - справа от панели
  - за логотипом
  - в пустых белых областях
- Убедиться, что взгляд остается на:
  - заголовке панели
  - форме входа
  - CTA-кнопке
- При необходимости сместить темные акценты ближе к краям сцены.
- Output / exit criterion:
  - фон дополняет композицию, а не тянет внимание на себя

### Phase 6: Проверить чистоту и “воздух” белой сцены
- После удаления полос отдельно проверить, не потерялся ли общий характер экрана.
- Если фон стал слишком пустым, добавить минимальный темный акцент:
  - по краям
  - в углах
  - крупным мягким силуэтом
- Не возвращать мелкий ритм; держать фон более крупным и спокойным.
- Output / exit criterion:
  - белый фон остается чистым, но не скучным

### Phase 7: Проверить адаптивность
- Проверить desktop-width, где белого пространства много.
- Проверить tablet/mobile, где крупные акценты могут вести себя лучше, чем частые полосы.
- Убедиться, что на маленьких экранах фон не выглядит тяжело и не создает грязные пятна вокруг формы.
- Output / exit criterion:
  - новый фон стабилен на разных ширинах

### Phase 8: Финальная визуальная полировка
- Сверить результат с общей эстетикой auth-экрана:
  - логотип слева сверху
  - темная центральная панель
  - белая карточка формы
  - строгая минималистичная типографика
- Проверить, что сцена ощущается “дороже” и спокойнее, чем с полосками.
- Output / exit criterion:
  - фон чистый, темный по настроению и визуально более элегантный

## Risks and Dependencies
- Риск: после удаления полос фон станет слишком пустым.
- Mitigation:
  - добавить один-два крупных темных акцента с мягкими краями вместо мелкого паттерна.

- Риск: темные формы будут конфликтовать с уже темной auth-панелью.
- Mitigation:
  - держать их менее контрастными и смещать в периферию сцены.

- Риск: новый фон получится слишком абстрактным и случайным.
- Mitigation:
  - использовать крупные спокойные формы, а не много мелких декоративных элементов.
