## Goal
Уменьшить composer-панель визуально по размеру, выровнять поле сообщения по вертикальному центру относительно кнопки-скрепки и сделать так, чтобы появляющаяся кнопка отправки тоже вставала по той же центральной линии, без ощущения смещения вниз или вверх.

## Assumptions
- Работа идет только в [conversation-view.tsx](C:\Users\User\Desktop\Project\apps\web\components\conversation-view.tsx).
- Нужно менять только верстку и визуальную геометрию composer, а не логику отправки.
- Скрепка слева остается круглой.
- Кнопка отправки остается условно появляющейся, но должна быть лучше выровнена по вертикали.
- Счетчик символов пока остается, если он не мешает выравниванию.

## Constraints
- Не ломать `Enter` / `Ctrl+Enter`.
- Не ломать attach flow и preview файла.
- Не возвращать удаленные helper-подписи.
- Сохранить текущие `data-testid`.

## Architecture / Approach
Проблема сейчас визуально состоит из двух частей:
- белая панель стала слишком высокой и “воздушной”
- textarea и send-button воспринимаются не по одной центральной оси со скрепкой

Значит решение нужно строить через:
- уменьшение высоты и внутренних отступов белой панели
- пересборку вертикального выравнивания ее внутренних элементов
- выравнивание send-button по той же опорной линии, что и круг со скрепкой

## Plan
### Phase 1: Зафиксировать текущую геометрию composer
- Проверить текущие размеры:
  - высота круга со скрепкой `h-12 w-12`
  - высота textarea `h-[56px] min-h-[56px]`
  - паддинги контейнера `pl-2.5 pr-4 py-2.5`
  - радиус панели `rounded-[30px]`
- Определить, что именно создает ощущение избыточной ширины/высоты:
  - большой внутренний `py`
  - слишком высокая textarea
  - нижняя зона со счетчиком
- Output / exit criterion:
  - понятна текущая опорная геометрия и основные источники “переразмера”

### Phase 2: Уменьшить визуальную высоту белой панели
- Снизить общую высоту composer за счет:
  - уменьшения `py` внешнего белого контейнера
  - уменьшения минимальной высоты textarea
  - при необходимости уменьшения радиуса панели
- Не делать панель слишком плоской; она должна оставаться удобной для набора.
- Проверить, что после уменьшения панель все еще комфортна для одной строки текста.
- Output / exit criterion:
  - composer заметно компактнее по высоте и не выглядит “надутым”

### Phase 3: Выровнять текст по центру относительно скрепки
- Сейчас скрепка воспринимается как центральный ориентир, а текст сидит чуть выше/ниже из-за высоты textarea и нижнего counter-row.
- Нужно:
  - уменьшить высоту textarea
  - пересобрать ее `py`
  - скорректировать расположение counter, чтобы основной текст визуально был на одной центральной оси со скрепкой
- При необходимости:
  - перенести counter чуть ниже, но не так, чтобы он смещал весь блок
  - или сделать текстовую область вертикально более нейтральной
- Output / exit criterion:
  - placeholder и вводимый текст визуально стоят на уровне центра скрепки

### Phase 4: Выровнять send-button по той же линии
- Появляющаяся кнопка отправки должна быть центрирована относительно той же оси, что:
  - скрепка
  - основная текстовая строка
- Проверить текущий контейнер:
  - `flex items-end gap-3`
  - `overflow-hidden ... w-12 opacity-100`
- Скорее всего нужно:
  - сменить общее выравнивание на `items-center`
  - или отдельно скорректировать align только для send-button wrapper
- Output / exit criterion:
  - кнопка отправки появляется ровно по центру и не “проваливается” вниз

### Phase 5: Пересобрать нижнюю зону со счетчиком
- После уменьшения панели проверить, не мешает ли текущий counter-row вертикальному балансу.
- Возможные варианты:
  - оставить только counter, но сделать его компактнее
  - уменьшить `mt` / `pb`
  - слегка изменить размер строки со счетчиком
- Смысл:
  - не позволять нижнему блоку визуально раздувать composer
- Output / exit criterion:
  - счетчик остается полезным, но не разрушает вертикальное центрирование

### Phase 6: Проверить состояния без send-button и с send-button
- Проверить пустое состояние:
  - send-button скрыта
  - composer и скрепка смотрятся ровно
- Проверить состояние с текстом:
  - send-button появляется
  - общая ось выравнивания не ломается
- При необходимости:
  - чуть подправить `gap` между белой панелью и send-button
- Output / exit criterion:
  - в обоих состояниях панель выглядит устойчиво и ровно

### Phase 7: Проверить attach и многострочный ввод
- После уменьшения панели проверить:
  - attach-only сценарий
  - текст + attach
  - многострочный ввод через `Ctrl+Enter`
- Убедиться, что:
  - textarea все еще может расти по высоте
  - панель не ломается при 2+ строках
  - preview файла остается корректным
- Output / exit criterion:
  - уменьшение панели не ухудшает реальные сценарии использования

### Phase 8: Финальная визуальная полировка
- После основной правки оценить:
  - не стала ли панель слишком тесной
  - не выглядит ли send-button слишком отдельно от composer
  - достаточно ли ровно стоят текст, скрепка и send-button на одной оси
- При необходимости сделать последнюю микроитерацию по:
  - `gap`
  - `py`
  - `textarea min-height`
  - `counter-row spacing`
- Output / exit criterion:
  - composer стал компактнее и визуально ровнее, без потери usability

## Risks and Dependencies
- Риск: слишком сильное уменьшение панели сделает набор тесным.
- Mitigation:
  - уменьшать высоту поэтапно и проверять многострочный ввод.

- Риск: выравнивание текста “по центру” визуально сломает counter-row.
- Mitigation:
  - корректировать не только textarea, но и нижний блок со счетчиком.

- Риск: send-button будет прыгать при появлении.
- Mitigation:
  - проверить и при необходимости подправить wrapper и общий `items-*` alignment.

- Риск: attach preview станет визуально несогласован с более компактным composer.
- Mitigation:
  - проверить preview после основной геометрической правки, но не менять его без необходимости.
