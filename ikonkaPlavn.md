# План реализации плавного появления сообщения/панели удаления

## Goal
Сделать появление кастомной панели подтверждения удаления визуально плавным: без резкого скачка в интерфейсе, с аккуратным fade/slide эффектом при открытии и закрытии.

## Assumptions
- Речь идет о кастомной confirm-панели удаления, которая уже реализована в `apps/web/components/confirm-dialog.tsx`.
- Меняем только frontend-анимацию и presentation layer.
- Логика удаления сообщения и чата уже работает и не должна меняться.
- Анимация должна быть легкой, быстрой и не мешать UX.

## Current State
Сейчас confirm-panel:
- открывается и исчезает сразу
- не имеет отдельной анимации overlay и самой панели
- функционально работает корректно, но визуально появляется слишком резко

## Approach
Сделать анимацию в два слоя:
1. затемнение фона появляется через мягкий fade
2. сама панель появляется через fade + небольшой slide/scale

При этом важно:
- не ломать accessibility
- не ломать тесты
- не зависеть от тяжелых animation libraries, если это можно решить обычным React + CSS

## Plan

### Phase 1: Зафиксировать желаемое поведение анимации
- Определить итоговую механику:
  - overlay: плавное изменение прозрачности
  - panel: плавное изменение opacity + небольшой сдвиг вверх/вниз или scale
- Выбрать короткую длительность:
  - примерно 160-220ms
- Проверить, что анимация должна быть одинаковой для:
  - удаления сообщения
  - удаления чата
- Output / exit criterion:
  - есть одна четкая animation model для confirm-dialog

### Phase 2: Подготовить компонент к анимированному mount/unmount
- Проверить текущую реализацию `ConfirmDialog` в `apps/web/components/confirm-dialog.tsx`.
- Если компонент сейчас просто не рендерится при `open = false`, добавить controlled transition-state:
  - `isMounted`
  - `isVisible`
  - таймер на завершение exit animation
- Цель:
  - при закрытии dialog не исчезает мгновенно
  - сначала проигрывает exit animation, потом размонтируется
- Output / exit criterion:
  - компонент умеет корректно переживать enter и exit animation

### Phase 3: Разделить анимацию overlay и panel
- В компоненте сделать отдельные transition-state классы для:
  - overlay
  - panel body
- Overlay:
  - `opacity-0 -> opacity-100`
- Panel:
  - `opacity-0 -> opacity-100`
  - `translate-y-2 -> translate-y-0` или `scale-[0.98] -> scale-100`
- Выбрать наиболее естественный вариант после визуальной проверки.
- Output / exit criterion:
  - фон и панель двигаются/появляются независимо и мягко

### Phase 4: Вынести transition-стили в понятную схему
- Решить, где лучше держать animation styles:
  - прямо в `clsx` внутри компонента
  - или как reusable utility-классы в `globals.css`
- Рекомендация:
  - если анимация нужна только confirm-dialog, можно оставить в компоненте
  - если стиль потенциально пригодится для других panel/modal, вынести utility классы
- Output / exit criterion:
  - animation code читаемый и не превращается в хаотичный набор классов

### Phase 5: Сохранить корректное поведение UX и accessibility
- Убедиться, что во время анимации:
  - `Escape` продолжает закрывать dialog
  - `onCancel` работает сразу
  - повторный confirm во время loading не ломает состояние
- Проверить, что focus не теряется во время enter/exit
- Если включен reduced motion, рассмотреть упрощение анимации:
  - либо сильно сократить движение
  - либо оставить только fade
- Output / exit criterion:
  - confirm-dialog остается удобным и доступным, а не только красивым

### Phase 6: Проверить влияние на тесты
- Текущие e2e уже ждут `confirm-dialog`.
- После добавления анимации убедиться, что:
  - dialog считается видимым в нужный момент
  - confirm button становится кликабельной без flaky поведения
- При необходимости в Playwright добавить чуть более устойчивое ожидание видимости/enable state.
- Output / exit criterion:
  - анимация не ломает стабильность тестов

### Phase 7: Прогнать визуальную проверку в обоих сценариях
- Проверить вручную:
  - удалить сообщение
  - удалить чат
- Убедиться, что:
  - панель появляется мягко
  - закрывается мягко
  - не выглядит медленно
  - не спорит с остальным интерфейсом
- Output / exit criterion:
  - UX удаления ощущается аккуратным и современным

- Проверить удаление чата

## Risks and Dependencies
- Риск: exit animation будет обрываться из-за мгновенного unmount.
- Mitigation:
  - использовать промежуточный mounted/visible state.

- Риск: анимация сделает e2e flaky.
- Mitigation:
  - оставить короткую длительность и проверить видимость/кликабельность dialog.

- Риск: слишком заметный slide будет выглядеть чужеродно.
- Mitigation:
  - сделать движение минимальным, а основной акцент оставить на fade.

- Риск: reduced motion не будет учтен.
- Mitigation:
  - добавить мягкий fallback без заметного движения.
