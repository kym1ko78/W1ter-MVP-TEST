## Objective
Убрать ручной resize у поля сообщения и сделать автоматический рост `textarea` по мере ввода текста, сохранив стартовую высоту, равную кнопке `Отправить`.

## Assumptions
- Поле должно расти автоматически по содержимому, без ручки resize в правом нижнем углу.
- Базовая высота должна остаться такой же, как сейчас у кнопки.
- Если текст станет слишком длинным, нужен верхний лимит роста, после которого включается внутренний скролл.
- Меняется только frontend UI/UX, без изменений backend.

## Current State
Сейчас в `apps/web/components/conversation-view.tsx`:
- `textarea` имеет фиксированную стартовую высоту `h-14`
- включен `resize-y`
- пользователь может тянуть поле вручную
- авто-роста по содержимому пока нет

## Implementation Plan

### Phase 1: Убрать ручной resize
- Удалить `resize-y` у `textarea`
- Задать `resize-none`, чтобы ручка изменения размера исчезла
- Проверить, что стартовая высота поля визуально не изменилась

Output / exit criterion:
- значок/ручка resize больше не отображается

### Phase 2: Добавить авто-рост textarea
- Добавить `ref` на `textarea`
- На каждом изменении текста:
- временно сбрасывать высоту до базовой
- измерять `scrollHeight`
- выставлять новую высоту по содержимому
- Сохранять минимальную высоту, равную высоте кнопки

Output / exit criterion:
- поле автоматически увеличивается вниз при наборе длинного текста

### Phase 3: Ограничить максимальный рост
- Выбрать максимальную высоту composer, например `160px` или `200px`
- Если `scrollHeight` превышает лимит:
- фиксировать высоту на максимуме
- включать внутренний `overflow-y-auto`
- Если текст снова становится короче:
- поле должно уменьшаться обратно

Output / exit criterion:
- поле растет плавно, но не ломает весь экран при длинном сообщении

### Phase 4: Синхронизировать кнопку и layout
- Проверить, как ведет себя кнопка, когда поле становится выше
- Оставить кнопку привязанной к верхнему краю composer-строки или к низу, в зависимости от текущего лучшего вида
- Проверить, что счетчик символов под полем не прыгает и не конфликтует с высотой

Output / exit criterion:
- рост поля не ломает сетку composer и не портит кнопку

### Phase 5: Проверить edge cases
- Пустое поле: высота как у кнопки
- 1 короткая строка: ничего не скачет
- 2-5 строк: textarea плавно растет
- Очень длинный текст: textarea упирается в максимум и начинает скроллиться внутри
- Очистка текста: textarea уменьшается обратно до стартовой высоты
- Отправка сообщения: после очистки поля высота возвращается к дефолту

Output / exit criterion:
- поведение стабильно и предсказуемо во всех типовых сценариях

## Risks and Dependencies
- Риск: при авто-росте textarea может “прыгать” на каждом символе.
- Митигация: сначала сбрасывать высоту, потом выставлять `scrollHeight`.
- Риск: после отправки поле может не возвращаться к стартовой высоте.
- Митигация: пересчитывать высоту и при `draft=""`.
- Риск: длинный текст может растянуть composer слишком сильно.
- Митигация: поставить верхний лимит высоты.
- Риск: на mobile автоподстройка может выглядеть иначе.
- Митигация: проверить вручную desktop + mobile width.
