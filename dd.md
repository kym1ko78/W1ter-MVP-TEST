## Goal
Надежно убрать встроенную лупу из поискового поля в левом sidebar, подтвердив, что удаляется именно тот SVG, который реально рендерится в живом DOM, и что после правки не остается лишнего отступа слева.

## Assumptions
- Лупа находится в блоке sidebar search рядом с кнопкой меню, внутри `chat-shell.tsx` или его актуального источника.
- Возможна одна из двух причин, почему иконка осталась видимой:
- правка была внесена не в тот актуальный JSX-фрагмент;
- dev-сервер показывал старую сборку или есть второй идентичный search block.
- Нужно не только убрать SVG, но и проверить, что `pl-11` или аналогичный левый padding тоже удален.

## Plan
### Phase 1: Подтвердить реальный источник иконки
- Найти точный JSX-фрагмент, который соответствует текущему DOM:
- `label.relative block min-w-0 flex-1`
- абсолютный `SearchIcon`
- `input[data-testid="global-search-input"]`
- `placeholder="Поиск"`
- Проверить, нет ли второго такого же блока в проекте, который тоже может рендериться.
- Если поиск по строке не дает единственного места, искать по `data-testid="global-search-input"` и по сочетанию `pl-11` + `SearchIcon`.
- Выходной результат фазы: один конкретный источник иконки или подтверждение, что есть дубликат.

### Phase 2: Сверить файл с актуальным состоянием
- Открыть текущий участок `chat-shell.tsx` вокруг `global-search-input`.
- Убедиться, что в файле реально еще присутствуют:
- `<SearchIcon ... absolute left-4 top-1/2 ... />`
- `pl-11`
- Если в файле их уже нет, значит проблема в другом месте:
- другой компонент;
- stale build;
- второй рендер того же sidebar.
- Выходной результат фазы: ясно, исправляем ли `chat-shell.tsx` или другой реальный источник.

### Phase 3: Удалить иконку и выровнять input
- Удалить JSX-узел с иконкой-лупой из актуального блока sidebar search.
- Заменить левый padding input на обычный проектный отступ:
- `px-4` или эквивалент;
- без резервирования места под абсолютный SVG.
- Проверить, что placeholder и текст поиска начинаются в правильной точке и не прилипают к границе.
- Выходной результат фазы: search field визуально чистый и без пустой зоны слева.

### Phase 4: Проверить, нет ли второй лупы в том же UX-потоке
- Повторно пробежать по проекту поиском по `SearchIcon` и absolute-position классам.
- Отдельно проверить sidebar menu / global search / header search, чтобы случайно не спутать:
- кнопку-лупу в header;
- декоративную лупу внутри input;
- иконки в search results.
- Выходной результат фазы: убрана именно внутренняя лупа поля, а не другая нужная иконка.

### Phase 5: Validation
- Прогнать `pnpm --filter @repo/web typecheck`.
- Прогнать `pnpm verify:web`.
- Перезагрузить dev-экран и проверить sidebar visually:
- лупы внутри input нет;
- input не смещен;
- поиск по-прежнему работает;
- menu button слева не затронута.
- Если в браузере иконка останется, дополнительно проверить dev cache / hot reload и фактический собранный JSX.
- Выходной результат фазы: проблема закрыта не только в коде, но и в реальном UI.

## Risks and Dependencies
- Риск: можно снова изменить не тот блок, если в кодовой базе есть похожий search input.
- Смягчение: ориентироваться по `data-testid="global-search-input"` и полному DOM-паттерну, который прислал пользователь.

- Риск: можно удалить SVG, но забыть убрать `pl-11`, и поле останется визуально кривым.
- Смягчение: всегда править SVG и padding в одной правке.

- Риск: dev-сервер может показывать устаревшее состояние после частичной hot reload-пересборки.
- Смягчение: после правки проверить сборку и при необходимости перезапустить dev, если UI не обновится.

## Next Actions
- Повторно открыть актуальный участок `chat-shell.tsx` по `global-search-input`.
- Удалить абсолютную лупу и заменить `pl-11` на нормальный нейтральный padding.
- Прогнать `typecheck`, `verify:web` и подтвердить визуально, что лупа действительно исчезла.
