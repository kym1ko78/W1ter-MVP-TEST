## Goal
Разместить темную auth-панель по центру всего экрана, а не только по центру правой колонки сетки.

## Assumptions
- Проблема сейчас в том, что auth-screen все еще живет в двухколоночной сетке, и темная панель центрируется только внутри правой части.
- Нужно сместить саму темную панель так, чтобы она воспринималась как центральный главный блок всей страницы.
- Логотип `W1ter` можно оставить, но он не должен мешать центрированию главной панели.
- Полосатый фон на белом пространстве можно сохранить, если он не мешает новой композиции.

## Constraints
- Не ломать логику `/login` и `/register`.
- Не ломать responsive behavior.
- Не терять текущую правую темную карточку и форму входа.
- Не делать layout “сломанным” на мобильных ширинах.

## Architecture / Approach
Проблема архитектурная: текущий `grid` с левой и правой колонкой искусственно удерживает темную панель справа.

Чтобы реально центрировать ее по всему блоку, нужно:
- перестроить верхний layout auth-screen
- отделить логотип от сеточной логики
- сделать темную панель главным центральным элементом страницы

Вероятный путь:
- убрать жесткую двухколоночную зависимость
- вынести логотип в отдельный абсолютный/overlay-слой
- центральную панель поместить в самостоятельный flex-контейнер по центру экрана

- Output / exit criterion:

### Phase 2: Выбрать новую layout-модель
- Вместо двухколоночной композиции выбрать модель:
  - full-screen stage
  - отдельно positioned logo
  - отдельно centered auth-card
- Это означает:
  - логотип не участвует в расчете центра основной панели
  - центральная карточка живет в собственном контейнере с `items-center justify-center`
- Output / exit criterion:
  - есть понятная схема, в которой центр панели считается от всего экрана, а не от колонки

### Phase 3: Вынести логотип из сеточной зависимости
- Переместить `W1ter` в отдельный слой:
  - либо absolute-позиционирование на белом фоне
  - либо отдельный overlay-контейнер
- Проверить, что логотип:
  - остается видимым
  - не толкает центральную панель
  - не ломает мобильную адаптацию
- Output / exit criterion:
  - логотип существует отдельно и больше не влияет на центрирование auth-card

### Phase 4: Перестроить контейнер темной панели в настоящий fullscreen-center
- Создать центральный контейнер для auth-card:
  - `min-h-screen`
  - `flex`
  - `items-center`
  - `justify-center`
- Поместить темную панель в этот контейнер как основной объект композиции.
- Проверить:
  - outer max-width
  - vertical alignment
  - достаточные safe paddings по краям
- Output / exit criterion:
  - темная панель располагается по центру всего экрана

### Phase 5: Сохранить внутреннюю композицию панели
- После внешнего перемещения проверить, как внутри панели выглядят:
  - заголовок
  - бейдж `Alfa`
  - белая form-card
- Возможно, придется скорректировать:
  - внутренние `mt`
  - `justify-center` / `justify-between`
  - общую высоту панели
- Output / exit criterion:
  - сама панель остается гармоничной после глобального центрирования

### Phase 6: Согласовать полосатый фон с новым центром
- После того как панель окажется в центре, проверить полосатый белый фон:
  - не отвлекает ли он
  - не создает ли асимметрию из-за логотипа слева
- При необходимости:
  - ослабить полоски
  - изменить их opacity
  - ограничить их только фоновым слоем
- Output / exit criterion:
  - центр композиции читается ясно, фон не спорит с ним

### Phase 7: Проверить `/login` и `/register`
- Убедиться, что новая fullscreen-centered схема одинаково хорошо работает:
  - для входа
  - для регистрации
- Проверить, не сломались ли:
  - ширины формы
  - вертикальные отступы
  - общий баланс экрана
- Output / exit criterion:
  - обе auth-страницы выглядят стабильно и центрировано

### Phase 8: Проверить мобильный и узкий viewport
- На меньших ширинах нужно убедиться, что:
  - центральная панель не выходит за экран
  - логотип не перекрывает форму
  - layout не теряет читабельность
- Возможно, понадобится:
  - отдельный mobile fallback без absolute-позиционирования логотипа
- Output / exit criterion:
  - экран остается корректным не только на desktop

## Risks and Dependencies
- Риск: логотип после выноса в overlay будет перекрывать форму.
- Mitigation:
  - использовать безопасное absolute-позиционирование и проверить mobile fallback.

- Риск: исчезнет визуальная структура экрана, если убрать grid слишком грубо.
- Mitigation:
  - оставить stage-обертку и фон, но развязать их от колоночного центрирования.

- Риск: центрирование панели по экрану потребует дополнительных правок внутренних отступов.
- Mitigation:
  - после внешнего repositioning отдельно проверить внутреннюю композицию панели.
